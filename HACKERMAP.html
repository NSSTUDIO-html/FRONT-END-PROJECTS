1. Foundations (must before anything else)

These fundamentals stay useful no matter which specialized path you follow.

  • Computer basics: filesystems, processes, services, shell (bash/powershell).
  • Networking: OSI model, TCP/UDP, IP, subnetting, routing, DNS, HTTP(S), TLS.
  • Operating systems: Deep Linux, Windows internals (registry, services, Event Logs).
  • Programming / scripting: Python (primary), Bash, PowerShell, JavaScript basics.
  • Databases: SQL basics and injection concepts.
  • Cryptography fundamentals: symmetric/asymmetric, hashing, TLS basics.

2. Core Domains & What to Learn in Each

Web Application Security

  • Skills: HTTP, sessions, auth, cookies, CSRF, XSS, SQLi, file upload bugs, RCE.
  • Tools: Burp Suite, OWASP ZAP, sqlmap, curl, Postman.
  • Resources: OWASP Top 10, Web Application Hacker’s Handbook.

Network Security & Wireless

  • Skills: packet capture/analysis, IDS/IPS, routing attacks, Wi‑Fi attacks, VPNs.
  • Tools: Wireshark, tcpdump, Nmap, Aircrack-ng, Bettercap, Netcat.

System / Host Security

  • Hardening, patching, EDR, privilege escalation, persistence.
  • Tools: Sysinternals (Windows), auditd, OSQuery, autoruns.

Offensive Security / Pentesting (Red Team)

  • Recon/OSINT, scanning, exploitation (in labs), pivoting, post-exploitation, reporting.
  • Tools: Kali, Metasploit (lab-only), proxy tools.

Defensive Security (Blue Team / SOC)

  • Alert triage, incident response, log analysis, endpoint detection, threat hunting.
  • Tools: SIEM (Splunk/ELK), EDR (CrowdStrike, Sentinel), Suricata, Zeek.

Incident Response & Forensics

  • Disk & memory forensics, timeline building, containment, evidence preservation.
  • Tools: Volatility, Autopsy, FTK Imager, SleuthKit.

Reverse Engineering & Malware Analysis

  • Assembly basics, static/dynamic analysis, unpacking.
  • Tools: Ghidra, IDA, x64dbg, OllyDbg.

Cloud Security

  • IAM, secure architecture, cloud-native threats, misconfigurations.
  • Tools: CloudTrail, ScoutSuite, Prowler; learn AWS/Azure/GCP basics.

Mobile & IoT Security

  • App hardening, reverse engineering APKs, firmware analysis, Bluetooth/BLE.
  • Tools: apktool, jadx, Frida, Burp for mobile proxies.

Identity & Access Management (IAM)

  • Auth flows, SSO, OAuth/OpenID, MFA concepts and defenses.

Threat Intelligence & OSINT

  • Mapping attacker TTPs, tracking leaks, OSINT tooling like Shodan & Maltego.

Governance, Risk & Compliance (GRC)

  • Policies, risk assessments, compliance frameworks (ISO27001, PCI, GDPR).

4. Safe Practice Platforms & Labs

  • TryHackMe (beginner → guided rooms)
  • Hack The Box (realistic boxes)
  • OverTheWire (war games)
  • VulnHub (downloadable VMs)
  • CTFTime (timed competitions)
  • Offensive Security labs (OSCP)
  • SANS NetWars (paid)

5. Books & Courses (high-value)

  • The Web Application Hacker’s Handbook
  • Hacking: The Art of Exploitation
  • Practical Malware Analysis
  • The Hacker Playbook series
  • Offensive Security PWK (OSCP)
  • SANS courses for IR/Forensics

6. Certifications (path suggestions)

  • Beginner: CompTIA Security+
  • Intermediate: CEH (conceptual), OSCP (hands-on)
  • Advanced / Specialized: OSCE, GIAC certs (GCIH, GCIA), CISSP (managerial)

7. Sample Learning Roadmaps (suggested timelines)

  • 0–3 months (foundations): Linux, networking, Python, basic web concepts, OverTheWire.
  • 3–6 months (practical skills): Nmap/Wireshark, Burp, TryHackMe beginner rooms, small Python projects.
  • 6–12 months (specialize): choose a track, HTB boxes, OSCP prep or Security+.
  • 12–24 months (professional): CTFs, build portfolio, internships, advanced certs.

8. Practical Projects & Portfolio Ideas

  • Home lab with vulnerable VMs (Metasploitable, DVWA).
  • Write-ups for HTB/CTF solves (host on GitHub).
  • Automate recon tasks with Python (subdomain discovery, port scanning).
  • Build small SIEM demo using ELK + Suricata.
  • Do an app-sec code review of an open-source project.

9. Daily / Weekly Practice Routine

  • Daily: 30–60 min reading + 30 min scripting practice.
  • Weekly: 1 HTB/THM lab or CTF challenge with a write-up.
  • Monthly: improve your lab or detection rules; join a community event.

10. Soft Skills & Important Non-technical Skills

  • Report writing & communication.
  • Legal & ethics: know your local laws; always get permission.
  • Teamwork & curiosity.

11. Career Paths & Roles

  • Junior SOC Analyst → Senior SOC → Threat Hunter.
  • Junior Pentester → Senior Pentester → Red Team Lead.
  • Incident Responder, AppSec Engineer, Security Architect, CISO.

12. Final Tips & Mindset

  • Focus on fundamentals (networking, OS, programming).
  • Practice legally — labs, bug bounties in scope, employer-approved tests.
  • Keep notes and publish write-ups — build a portfolio.
